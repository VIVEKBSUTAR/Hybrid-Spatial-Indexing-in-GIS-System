# Hybrid Spatial Indexing in GIS System

## Overview

This project is a Mini Geographic Information System (GIS) that implements hybrid spatial indexing using Grid-based indexing and Quadtree data structures. The goal is to efficiently store, organize, and query spatial data points.

The system demonstrates how advanced data structures improve the performance of spatial queries compared to naive approaches.

---

## Key Features

* Efficient storage of spatial data points
* Grid-based and Octree-based 3D spatial indexing
* Quadtree-based 2D hierarchical spatial indexing
* Fast query processing for spatial operations (Range, NN, KNN)
* Hybrid backend seamlessly switching between 2D and 3D spatial contexts (e.g., flat terrain vs. altitude-aware queries)
* Interactive WebGL 3D visualization and 2D mapping interface
* Modular architecture using C algorithms and Python API routing

---

## System Architecture

The project follows a multi-layer architecture:

### 1. Core Processing Layer (C)

* Implements core spatial data structures for high performance
* Handles indexing and mathematical query processing
* Separated into **3D Octree Engine** (`main_octree.exe`) and **2D Quadtree Engine** (`main_quadtree.exe`) to precisely handle different geographical search models.

### 2. Server Layer (Python)

* Acts as a bridge between frontend and backend
* Executes the compiled C program
* Handles request-response communication

### 3. Presentation Layer (HTML)

* Provides an advanced **Hybrid GIS** interface (`application.html`) for 2D terrain mapping and interaction
* Provides a **MiniGIS 3D Engine** (`index.html`) using `Three.js` for dynamic point-cloud and spatial boundary visualization
* Allows users to input queries visually and toggle between index engines

## Setup and Installation

### 1. Clone the repository

```
git clone https://github.com/VIVEKBSUTAR/Hybrid-Spatial-Indexing-in-GIS-System.git
cd Hybrid-Spatial-Indexing-in-GIS-System
```

### 2. Compile the C backend engines

```
gcc -O2 -o main_octree.exe main_octree.c octree.c grid.c queries.c -lm
gcc -O2 -o main_quadtree.exe main_quadtree.c quadtree.c queries_2d.c -lm
```

### 3. Run the Python application server

```
python app_server.py
```

### 4. Open the frontend

Open `index.html` in a browser.

## How It Works

1. Spatial data and terrain mapping are dynamically generated by the frontend.
2. Data is written to disk and instantly indexed using:
   * 3D Octree structures for volumetric data
   * 2D Quadtree structures for flat terrains
3. Users click on the map to issue point-based queries.
4. The JavaScript frontend determines the spatial context (e.g. clicking on elevated terrain ignores Z-depth, clicking on the ground considers it).
5. Python server routes the request to the appropriate C engine.
6. The C algorithms process KNN or Range queries and return results for real-time visualization.
